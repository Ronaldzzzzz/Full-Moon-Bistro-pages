無伺服器 + 免費資料庫方案

靜態託管（GitHub / Cloudflare Pages）＋ 多人後台協作

🔥 方案 A — Firebase (Firestore + Auth)
✓ 即時同步 (Realtime)
✓ 內建 Auth（Google 登入）
✓ 免費額度充裕
✓ 成熟生態、文件完整
✗ 依賴 Google
✗ NoSQL，非關聯式
架構：React → Firebase SDK → Firestore DB

Firebase

業界最成熟的 BaaS。前端直接呼叫 Firebase SDK，無需後端程式。免費 Spark Plan 對 RP 小圈子完全足夠。推薦首選。

⚡ 方案 B — Supabase
✓ 真正的 PostgreSQL
✓ 內建 Auth + Row Level Security
✓ 開源可自部署
✓ 自動生成 REST/GraphQL API
✗ 免費專案閒置7天會暫停
~ 相對 Firebase 較新
架構：React → Supabase JS Client → PostgreSQL

Supabase

開源版 Firebase，底層是 PostgreSQL（關聯式 DB）。SQL 查詢更直覺，但免費專案若長時間無人使用會進入休眠，需定期喚醒。

☁️ 方案 C — Cloudflare Pages + D1 + Workers
✓ 全在 Cloudflare 生態內
✓ D1 是 SQLite，免費額度大
✓ Workers = 邊緣函數 API
✓ 無休眠問題
✗ 需自行撰寫 API（Workers）
✗ 開發體驗較複雜
架構：React → Cloudflare Workers API → D1 SQLite

Cloudflare D1 + Workers

若選 Cloudflare Pages 部署，可搭配 D1（SQLite）和 Workers（無伺服器函數）。全站都在 CF 生態，延遲極低，但需要自己寫後端 API 邏輯，開發成本較高。

我的推薦：方案 A（Firebase）
對 RP 小圈子而言，Firebase 免費額度完全夠用，Auth 開箱即用（可設定只有特定 Google 帳號能登入後台），且與 GitHub Pages / Cloudflare Pages 完美搭配，前端直接呼叫 SDK 即可。
