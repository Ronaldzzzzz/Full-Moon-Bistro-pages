# Global AI Agent Instructions

## 0. 核心準則
- **本檔案為跨平台通用指令。無論使用何種模型或 CLI，必須嚴格遵守以下準則。**
- **未經允許，禁止修改此檔案。**
- **協作聲明**：你將與其他 AI Agent（如 Gemini, Claude, Copilot 等）共同實作此專案。你的產出必須具備高度的可讀性與結構化，以便其他 Agent 銜接。

## 1. 語言規範
- **全程使用中文回覆。**
- 僅在專有名詞、程式碼註解、技術術語時使用英文。

## 2. 初始化任務 (Initial Sync)
- **啟動必讀**：請讀取當前目錄下的 **PROPOSAL.md**。
- **完善計畫**：基於 PROPOSAL 中的願景與需求，完善並拆解 **PLAN.md** 中的實作細節，確保計畫具備可執行性。

## 3. Skill 使用時機 (動態權重)
AI 應根據任務複雜度自行評估是否啟動 Karpathy + Superpowers 規範：
- **工業模式 (啟動)**：涉及「核心邏輯、架構變動、複雜演算法、多檔案連動」時。
- **輕量模式 (停用)**：涉及「簡單文案修改 (i18n)、UI 微調、單一變數命名、已知錯誤快速修復」時。
- **原則**：簡單任務應以「效率」為先，無需更新 PLAN/PROGRESS 或執行 TDD。

## 4. 開發心法 (工業模式)
- **Minimalism (Karpathy)**: 遵循外科手術式修改，嚴禁無關重構，保持 Diff 乾淨。
- **Thinking**: 編碼前先陳述邏輯假設與設計思路（協助其他 Agent 理解意圖）。
- **TDD (Superpowers)**: 優先規劃測試，確保功能可驗證。
- **Atomic Tasks**: 將任務拆解為微小、可管理的單元執行。

## 5. 互動機制
- **禁止瞎猜**: 遇指令不明確、邏輯矛盾或潛在風險時，**必須主動詢問**。
- **主動終止**: 若測試失敗兩次以上，停止嘗試並列出「已知事實」與「困惑點」尋求人類協助。

## 6. 專案追蹤 (工業模式下執行)
若資料夾內缺失以下檔案，請在啟動工業模式時創建：

### 6.1 PLAN.md (專案藍圖)
- 基於 PROPOSAL.md 維護詳細實作計畫。完成後註記 `[DONE]`。

### 6.2 PROGRESS.md (進度日誌)
- **交接標準**：極簡紀錄「當前進度」、「受影響檔案」、「改動目的」。
- **用意**：確保不同 AI 代理在切換時能實現零損耗、無縫接力。
